#### 동기
- 빗물받이 덮개로 담배꽁초나 낙엽 쓰레기등이 유입되면 막힘 현상에 의해 도시 침수 피해를 유발

#### 목표
- CNN,LSTM 기반 하수구 쓰레기 막힘 예측
- Django REST Framework 기반 API 서버와 AI 모델 통합
- MySQL 기반 데이터 구조 설계 및 실시간 모니터링 환경 구축

## 기술 문제 해결 경험 및 직무 적용 역랑

■ 임베디드 환경 내 실시간 데이터 처리 최적화
- 문제: 라즈베리 파이 리소스 한계로 인해 CNN-LSTM 모델 구동 시 데이터 처리 지연 발생
- 분석: 연속적인 이미지 및 센서 데이터 입력 시 메모리 점유율 급증 확인
- 조치: 이미지 프레임 스킵 기법 적용 및 모델 경량화, 수치 데이터와 결합하여 연산 부하 감소
- 결과: 평균 응답 속도 0.8초 이내 유지 및 10시간 이상 연속 가동 안정성 확보
- 직무 적용점: 저사양 하드웨어 환경에서도 시스템 Latency를 최소화하는 최적화 설계 역량 확보

■ DB 병목 및 데이터 구조 개선
- 문제: 센서값(온습도/미세먼지) 및 쓰레기 수위 데이터의 잦은 POST 요청으로 인한 조회 속도 저하
- 분석: 단일 테이블 내 대량 로그 적재로 인한 Index Full Scan 및 JOIN 병목 확인
-  조치: 관리 정보와 센서 로그 데이터 분리 설계, Django ORM 최적화 및 타임스탬프 인덱스 추가
- 결과: 조회 속도 약 40% 개선 및 데이터 처리 효율성 증대

직무 적용점: 데이터 흐름 기반 DB 구조 설계 및 성능 최적화 경험 확보

■ 시스템 통합 설계 및 파이프라인 구축
- 내용: 라즈베리 파이(센서/영상) → CNN-LSTM 모델 → Django API 서버 → React 대시보드 구조 설계
- 해결: 쓰레기 높이 20cm 필터링 로직을 API 레벨에서 구현하여 유효 데이터 선별 및 위험 판별 자동화
- 결과: 2024 캡스톤디자인 경진대회 대상 수상 (시스템 안정성 및 완성도 인정)
- 직무 적용점: AI-Backend 통합 아키텍처 설계 및 전체 시스템의 에러를 조율하는 컨트롤 타워 역량 보유

## 💡 Lessons Learned
하드웨어-AI-백엔드가 결합된 복합 시스템을 개발하며, 백엔드 개발자가 단순히 데이터만 전달하는 것이 아니라 전체 시스템의 지연(Latency)과 에러를 조율하는 컨트롤 타워 역할을 해야 함을 깊이 체득했습니다.
